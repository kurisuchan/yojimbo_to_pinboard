Python-Pinboard is an open source Python module to access the contents of a Pinboard account via the Pinboard API (which is based on the del.icio.us API).

Notes on Documentation

Attribute and method examples assume the existence of a PinboardAccount variable p which could have been created by the following code:

import delicious
p = pinboard.open("username", "password")

Brief Overview

What is Python-Pinboard? Why this implementation?

Python-Pinboard is a translation of Paul Mucur's Python module for accessing the del.icio.us API, Python-Delicious, for use with Pinboard.

Python-Pinboard is an open source Python module to access the contents of a Pinboard account.

Objects

PinboardAccount

This is the only object in the module and is the one that does all of the work. It represents a single Pinboard account and contains all of the attributes and methods defined in this documentation.

Two arguments MUST be specified:

username
The username for the Pinboard account.
password
The password for the Pinboard account.

As well as the usual Python syntax for creating a new object, two methods, open and connect, can be used like so:

import pinboard
	p = pinboard.PinboardAccount("username", "password")
	p = pinboard.open("username", "password")
	p = pinboard.connect("username", "password")

Attributes

As the module aims to represent a Pinboard account as a Python dictionary, data from your account is most easily accessed using the dictionary syntax of object["key"].

Important implementation note: In order to save repeatedly making requests to the Pinboard servers, most of the following attributes are only loaded upon request. By default, a PinboardAccount object only contains the last update information and the headers returned by the server. When some data is loaded into an attribute, simply using the usual dictionary syntax will immediately return whatever has been stored without requesting any information from Pinboard at all: this means that if the posts method has been used to download only a selected few posts, simply calling p["posts"] (where p is your PinboardAccount object) will only return those selected posts: not every single one of your posts as you might have expected. To work around this, simply use the posts method rather than the attribute when necessary.

Bundles

p["bundles"]

This attribute contains a list of all of the bundles in your Pinboard account.

Dates

p["dates"]

This attribute contains a list of all of the dates on which you have made posts to Pinboard. Each element in this list is a dictionary of the following form:

{u'count': 2,
	u'date': u'2005-05-08',
	u'date_parsed': (2005, 5, 8, 0, 0, 0, 6, 128, -1)}

The anatomy of the above dictionary is as follows:

count
The number of posts made on that date.
date
The exact string representing that date returned by Pinboard.
date_parsed
The above date parsed into a Python time tuple of nine integers.

Headers

This attribute is a dictionary of the headers returned by the Pinboard servers upon your last request and is of the following form:

{'connection': 'close',
 'content-encoding': 'gzip',
 'content-length': '97',
 'content-type': 'text/html',
 'date': 'Thu, 23 Dec 2010 06:35:55 GMT',
 'server': 'Apache/2.2.14 (Ubuntu)',
 'vary': 'Accept-Encoding'}

Tags

p["tags"]

A list of every tag in your Pinboard account. Each element is a dictionary:

{u'count': 1, u'name': u'aggregation'}

An anatomy of the above dictionary is as follows:

count
The number of posts associated with this tag.
name
The name of this tag.

Methods

As well as being a dictionary, the PinboardAccount object supports several methods primarily for more refined query-based retrieval of posts and dates and the manipulation of content on the server (such as adding, editing and deleting posts).

Bundles

p.bundles()

Use the attribute instead.

Dates

p.dates(tag="computing")

Has a single argument, a string tag, which will refine results so that only dates on which a post associated with the specified tag was made are returned.

Posts

p.posts(tag="computing", date="2005-05-07", count=5)

This is the most flexible method in the module and allows you to refine which posts you would like to retrieve from Pinboard. The arguments are as follows:

tag
A string specifying with which tag posts should be associated.
date
A string, tuple, date or datetime specifying on which date the posts should have been made.
count
The number of posts to return.

If no arguments are specified, this method will return all posts from Pinboard: this is highly discouraged as it places a large load on the Pinboard servers. However, the module will prevent you from repeatedly downloading all posts unnecessarily as it internally will check if any changes have been made and, if not, not make a new request.

Tags

p.tags()

Use the attribute instead.

Add (and Edit) Posts

p.add(url="http://example.org/", 
		description="Example",
		extended="An example site",
		tags=('computing', 'examples'),
		date=(2010, 12, 22))

This method will add new posts to Pinboard. Its arguments are as follows:

url
Required. The URL of the page you are bookmarking.
description
Required. A brief description for the post.
tags
A string, tuple or list of tags to associate this post with.
date
A string, tuple, list, date or datetime on which this post should be registered as made.

Bundle Tags

p.bundle(bundle="writing",
	tags=("blogging", "journalism"))

This method creates a new bundle from some existing tags. Its arguments are as follows:

bundle
Required. The name of the bundle you are creating.
tags
Required. A list or tuple of the tag names (as strings) that you wish to put inside this bundle.

Delete Posts

p.delete(url="http://example.org/")

Delete the post identified by the given URL, url.

Delete Bundles

p.delete_bundle(name="writing")

Delete the bundle identified by the given name, name.

Rename Tags

p.rename(old="blogging", new="weblogging")

Rename the tag old to new.
